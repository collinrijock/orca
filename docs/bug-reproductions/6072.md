# Issue #6072 — Mobile keeps showing old agent rows after terminals are closed

**Status:** REPRODUCED

## Summary

`OrcaRuntimeService.getWorktreePs()` (the projection that backs Orca Mobile's
sidebar and `orca worktree ps --json`) mixes live runtime state with persisted
session/hook state. After the last PTY for a worktree is gone, the summary can
still report:

1. a non-zero `liveTerminalCount`, sourced from persisted `tabsByWorktree`, and
2. finished (`done`) agent rows in `summary.agents`, sourced from the hydrated
   agent-hook snapshot,

even though there is no live terminal or pane. This matches the desktop-CLI
half of the report confirmed by @ericjung on v1.4.144 and the mobile symptom in
the original issue. PR #9053 (referenced in a comment) is not present in the
current tree, and the behavior still reproduces.

## Root cause

`src/main/runtime/orca-runtime.ts`

1. **Stale `liveTerminalCount`** — `getWorktreePs()` folds saved session tabs
   into the live count with `Math.max`, so any persisted tab keeps the count
   non-zero regardless of whether a PTY/leaf/process is live:

   ```
   src/main/runtime/orca-runtime.ts:12325
   summary.liveTerminalCount = Math.max(summary.liveTerminalCount, tabs.length)
   ```

2. **Stale `done` agent rows** — `attachAgentRowsToSummaries()` buckets every
   hook/OSC snapshot row by `mirroredWorktreeIdByTabId.get(tabId) ?? src.worktreeId`
   and assigns them to `summary.agents` without checking that the pane/tab still
   exists. Since `last-status.json` hydrates entries for days, a finished agent
   whose tab was closed still shows up. The only freshness gate
   (`isFreshNonDoneAgentStatus`, L12525) affects `hasHostSidebarActivity`/`status`,
   **not** whether the row is included in `summary.agents`:

   ```
   src/main/runtime/orca-runtime.ts:12403  attachAgentRowsToSummaries(...)
   src/main/runtime/orca-runtime.ts:12518  summary.agents = rows   (no tab-existence filter)
   ```

## Manual repro (from the issue)

1. Run a few agent terminals in a worktree; let them finish, then close the tabs.
2. Open Orca Mobile (or run the CLI on the host).
3. Compare with `orca worktree ps --json` and
   `orca terminal list --worktree 'id:<worktreeId>' --json`.
4. `worktree ps` still shows old `Done` agents and a non-zero `liveTerminalCount`
   with no matching live terminals.

## Fix direction

- Distinguish `savedTabCount` from `liveTerminalCount`; do not raise the live
  count from persisted tabs that have no backing live PTY/process (or scope the
  raise to tabs whose `ptyId` resolves to a connected PTY).
- In `attachAgentRowsToSummaries`, drop (or don't surface as current) `done`/
  stale rows whose pane/tab no longer exists in the current session, and clear
  hook status for a tab's pane keys when `session.tabs.close` runs.
