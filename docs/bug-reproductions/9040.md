# Issue #9040 — No processing/thinking indicators in left nav

**Status:** PARTIAL (derivation-level root cause proven & pinned with a passing unit test; full live-app path not exercised. A fix PR — #9048 — is already open and actively worked.)

## Summary
Since ~v1.4.141 the sidebar worktree cards stopped showing the "working"/thinking
spinner while an agent is still running, even though the amber "done" bell still
appears when it finishes. Reported over Discord against 1.4.141.

The sidebar dot is derived from the agent-status `state` stored per pane. The
shared status normalizer keeps a lead payload at `state: 'done'` even when that
payload still carries a **live working subagent**. Any Orca-compatible status
producer that publishes lead completion while a child is still running (OpenCode,
MiMo, Copilot, or a custom/relay producer that sends normalized child snapshots)
therefore lands in the store as `done`. The renderer working-status derivation
reads only `entry.state`, so the dot resolves to `done` (bell) instead of
`working` (spinner) — exactly the reported "no thinking indicator" symptom.

## Root cause
`src/shared/agent-status-types.ts`, `normalizeAgentStatusObject` (lines ~356–396).
It validates `subagents` and returns them, but never gates the pane state up to
`working` when a child is still working:

```ts
const state = obj.state          // stays 'done'
...
return { state: state as AgentStatusState, ..., subagents: normalizeSubagentsField(obj.subagents) }
```

There is Claude-specific done-gating in `src/shared/agent-hook-listener.ts`
(`claudeRosterHasWorkingSubagent`, lines ~2549/2572/2722), but it is provider-
specific — it does not cover generic/relay/custom producers whose payloads flow
straight through `normalizeAgentStatusPayload`. The open fix PR #9048 adds the
generic gate at the normalizer:

```ts
const hasWorkingSubagent = subagents?.some(({ state }) => state === 'working') === true
const state = rawState === 'done' && hasWorkingSubagent ? 'working' : rawState
```

Once the normalized state is `done`, the renderer derivation is faithful but
powerless: `selectWorktreeAgentActivitySummary`
(`src/renderer/src/components/sidebar/worktree-agent-activity-summary.ts`) sets
`hasLiveDone` (not `hasLiveWorking`), and `resolveWorktreeStatus`
(`src/renderer/src/lib/worktree-status.ts`) returns `done`.

## Automated evidence
Re-run:

```
pnpm exec vitest run --config config/vitest.config.ts src/renderer/src/components/sidebar/repro-9040-working-child-done-gate.test.ts
```

Result: 3 passed. The test imports the real product modules
(`normalizeAgentStatusPayload`, `selectWorktreeAgentActivitySummary`,
`resolveWorktreeStatus`) and pins the current behavior:
- A `{ state: 'done', subagents: [{ state: 'working' }] }` payload normalizes to
  `state: 'done'` (BUG; correct = `working`).
- Fed through the sidebar derivation, the worktree dot resolves to `done`
  (bell) instead of `working` (spinner).
- A control asserts the derivation itself is correct when the state is `working`
  — proving the loss happens upstream at normalization.

## Manual repro (app steps)
1. Run an Orca-compatible agent that spawns a background child/reviewer and
   reports lead completion while the child keeps working (e.g. OpenCode/MiMo, or
   a relay/custom producer sending normalized child snapshots).
2. Watch the left-nav worktree card while the child is still working.
   Observed: no spinner; the done/bell state shows. Expected: yellow working
   spinner until all children finish.

## Fix direction
Gate the normalized pane state up to `working` whenever a live working subagent
is present, at the shared normalizer so it covers every provider (as PR #9048
does). Keep the child snapshots so the roster/child rows still render.

## Key files
- `src/shared/agent-status-types.ts` (`normalizeAgentStatusObject`) — the missing generic done→working gate
- `src/shared/agent-hook-listener.ts` — existing Claude-only gating (`claudeRosterHasWorkingSubagent`)
- `src/renderer/src/components/sidebar/worktree-agent-activity-summary.ts` — derives `hasLiveWorking`/`hasLiveDone`
- `src/renderer/src/lib/worktree-status.ts` (`resolveWorktreeStatus`) — maps state to the sidebar dot
- `src/renderer/src/components/sidebar/StatusIndicator.tsx` — renders the working spinner vs done dot
