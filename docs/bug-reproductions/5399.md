# Issue #5399: Limits always shows 5h regardless of reset time

**Status:** REPRODUCED

## Summary
The collapsed usage/limits bar in the bottom-left status bar renders a hardcoded
`5h` label for the Claude session window. It never reflects the actual reset
time. When the session actually resets in, say, 1h20m, the collapsed bar still
shows `5h`, while the expanded popover (opened on click) shows the correct
`Resets in 1h 20m`. The two surfaces disagree because they compute the label
from different data.

## Root cause
- Collapsed bar: `InlineUsageBars` builds the session window label as a static
  string, ignoring `limits.session.resetsAt`:
  - `src/renderer/src/components/status-bar/StatusBar.tsx:987`
    ```ts
    label: translate('auto.components.status.bar.StatusBar.d79c3362c4', '5h')
    ```
  The `5h` comes from the window *length* (300 minutes), not the time-until-reset.
- Expanded popover: computes a live countdown from `resetsAt`:
  - `src/renderer/src/components/status-bar/tooltip.tsx:284`
    ```ts
    const resetLabel = w.resetsAt ? formatResetCountdown(w.resetsAt - Date.now()) : null
    ```
  - `formatResetCountdown` lives in `src/shared/rate-limit-reset-format.ts:29`
    and returns e.g. `Resets in 1h 20m`.

So the collapsed label is a fixed window-length string and is never recomputed
from the reset timestamp that the popover uses.

## Automated evidence
A repro unit test renders the real `InlineUsageBars` product component with a
session window whose `resetsAt` is ~1h20m in the future (window length still
300 min). It asserts the collapsed markup contains the static `32% used 5h` and
does NOT contain the real `1h 20m` reset time that `formatResetCountdown`
produces for the very same window.

Re-run:
```
pnpm exec vitest run --config config/vitest.config.ts src/renderer/src/components/status-bar/repro-5399-collapsed-reset-label.test.tsx
```
Result: passes (green) on the current tree, pinning the buggy static `5h` label.

## Manual repro
1. Sign in to Claude with a session whose 5h window is partway through (reset
   less than 5h away).
2. Look at the collapsed limits bar in the bottom-left status bar: it shows
   `NN% used 5h`.
3. Click it to open the popover: it shows `Resets in 1h 20m` (the true value).
4. The collapsed `5h` never changes as the reset approaches.

## Fix direction
Derive the collapsed session label from the reset time when available instead of
the hardcoded `5h`. Reuse the shared `formatResetDuration`/`formatResetCountdown`
helpers (`src/shared/rate-limit-reset-format.ts`) so the collapsed bar and the
popover stay consistent — e.g. show `formatResetDuration(session.resetsAt - Date.now())`
(`1h 20m`) and only fall back to a window-length label (`5h`) when `resetsAt`
is null.
