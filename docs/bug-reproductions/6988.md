# Issue #6988 — System SSH probe fails on GitHub with "Invalid command: echo ORCA-SYSTEM-SSH-OK"

**Status:** REPRODUCED (logic-level, via unit test that imports the real product code)

## Summary

Orca's system-SSH connectivity probe runs the shell command `echo ORCA-SYSTEM-SSH-OK`
on the remote host and treats the connection as reachable only when the remote
exits 0 AND stdout contains the marker. Hosts that run a restricted git-shell —
most notably `git@github.com` — reject any non-git command with
`Invalid command: echo ORCA-SYSTEM-SSH-OK` and exit non-zero. The probe therefore
reports `System SSH probe failed (exit 1)` even though the host authenticates
successfully and serves git over SSH (ls-remote/clone/push all work).

## Root cause

`src/main/ssh/ssh-connection.ts:842` — the probe spawns a bare shell command:

```ts
const channel = this.spawnTrackedSystemSshCommand('echo ORCA-SYSTEM-SSH-OK', {
  wrapCommand: false
})
```

`src/main/ssh/ssh-connection.ts:881` — success requires exit 0 and the echoed marker:

```ts
if (code !== 0 || !stdout.includes('ORCA-SYSTEM-SSH-OK')) {
  reject(new Error(`System SSH probe failed${...}. stderr: ${stderr.trim()}`))
```

A restricted shell (GitHub's git-shell allows only `git-upload-pack`,
`git-receive-pack`, `git-upload-archive`) rejects `echo ...`, so `code === 1` and
stdout is empty → the probe throws, even though authentication succeeded.

## Automated evidence

Re-run command:

```
pnpm exec vitest run --config config/vitest.config.ts src/main/ssh/repro-6988-github-restricted-shell-probe.test.ts
```

The test (`src/main/ssh/repro-6988-github-restricted-shell-probe.test.ts`) constructs
a real `SshConnection` for `git@github.com`, mocks the system-ssh channel to emit
GitHub's exact `Invalid command: echo ORCA-SYSTEM-SSH-OK` stderr with exit 1, then
drives the real `doSystemSshProbe`. It PASSES on the current tree while asserting:

- the probe rejects with `System SSH probe failed (exit 1)`;
- the failure reason is verbatim `Invalid command: echo ORCA-SYSTEM-SSH-OK`;
- the command actually sent is the bare shell `echo ORCA-SYSTEM-SSH-OK`;
- the connection is left un-connected.

Result: `Test Files 1 passed (1)` / `Tests 1 passed (1)`.

## Manual repro (requires a GitHub SSH key configured)

```
$ ssh git@github.com "echo ORCA-SYSTEM-SSH-OK"
Invalid command: echo ORCA-SYSTEM-SSH-OK
...
$ echo $?      # non-zero
$ ssh -T git@github.com   # succeeds: "Hi <user>! You've successfully authenticated..."
```

## Fix direction

Replace the shell-`echo` probe with a command a restricted shell accepts:

- Option A: `ssh -T` (no remote command). Auth-success can be detected from the
  stderr banner even though exit code may be 1.
- Option B: a git-protocol probe (e.g. `git-upload-pack`/`git-receive-pack`), which
  git-shell permits.

Either way, do not treat a host that authenticated successfully but refused a shell
command as unreachable.
