# Issue #8075 — Markdown editor does not support numeric superscript footnotes

**Status:** NOT REPRODUCED (fixed in current tree)

## Summary

The issue reports that inline HTML footnote syntax like
`<sup><a href="https://www.sci-gz.com/jtyw/jr/226.html">[12]</a></sup>` is not
supported in Orca's Markdown editor. On the current tree this **is** supported:
the rich-Markdown (WYSIWYG) editor renders the fragment as a first-class compact
citation atom, and the read-only Markdown preview also renders `<sup>`/`<a>` HTML.

The report predates the fix. PR #8307 (`feat(markdown): support HTML superscript
links`, merged 2026-07-11) added a dedicated rich-Markdown node for exactly this
syntax and enabled it in production.

## Root cause (of the original report — now resolved)

Before PR #8307, the rich-Markdown editor had no node for narrow
`<sup><a ...>label</a></sup>` citations, so the fragment fell through as inert
text and was not rendered as a footnote.

The fix lives in:
- `src/renderer/src/components/editor/rich-markdown-html-superscript-link.ts` — the
  Tiptap node that matches, renders, and round-trips the citation (parseMarkdown
  at line 61, renderHTML at line 80, live node view at line 100).
- `src/renderer/src/components/editor/rich-markdown-html-superscript-link-source.ts:35`
  — `matchHtmlSuperscriptLinkSource`, the byte-exact `<sup><a>…</a></sup>` matcher.
- Production wiring enables it: `useRichMarkdownEditorInstance.ts:15` and
  `rich-markdown-editor-config.ts:132` pass `htmlSuperscriptLinks: true`.

The read-only preview path already allowed the tags before the report:
`src/renderer/src/components/editor/MarkdownPreview.tsx:295` keeps `sub`/`sup` in
the rehype-sanitize schema, combined with `rehypeRaw` for inline HTML.

## Manual repro (app steps)

1. Open a `.md` file in Orca and switch to the rich Markdown editor.
2. Type/paste `<sup><a href="https://example.com">[12]</a></sup>`.
3. Current behavior: it renders as a compact `[12]` superscript citation (with
   Open/Copy affordances), not as raw text. The Markdown preview renders it too.

## Fix direction

None required — already implemented by PR #8307. If a regression appears, keep the
`htmlSuperscriptLinks` flag enabled in the production editor wiring and preserve
the `sub`/`sup` entries in the `MarkdownPreview` sanitize schema.
