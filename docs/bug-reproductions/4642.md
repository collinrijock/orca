# Issue #4642: `npx skills add --global` silently fails the PromptScript target but exits 0

**Status:** REPRODUCED

## Summary
Orca generates its skill-install command with a hardcoded `--global` flag. PromptScript — the only skill target Orca actually reads — cannot install globally, so that target is skipped. The `skills` CLI still exits 0 with a "✓ Installed 1 skill" banner (most of the ~17 universal targets succeed), so the user sees success but the skill never appears in Orca. The PromptScript failure is printed as an easy-to-miss footnote.

## Root cause
`src/shared/agent-feature-install-commands.ts:15` — `buildAgentFeatureSkillInstallCommand` appends `--global` to every generated command:

```ts
return `npx skills add ${ORCA_SKILLS_REPOSITORY_URL} --skill ${skillNames.join(' ')} --global`
```

Every exported install constant (`ORCHESTRATION_SKILL_INSTALL_COMMAND`, `COMPUTER_USE_SKILL_INSTALL_COMMAND`, etc.) is built from this function, so the whole UI surfaces the `--global` command. Because PromptScript rejects global installs, Orca's discovery (which reads project/home skill sources) never sees the skill.

## Automated evidence
A unit test imports the real product module and pins the buggy `--global` output:

```
pnpm exec vitest run --config config/vitest.config.ts src/shared/repro-4642-global-skill-install.test.ts
```

Result: 2 passed — the generated command is exactly `npx skills add https://github.com/stablyai/orca --skill orchestration --global`, and all exported install constants contain `--global`. Assertions in the test mark which lines encode the bug and what the correct (non-`--global`, project-scoped `-y`) command should be.

## Manual repro (from the issue)
1. Run `npx skills add https://github.com/stablyai/orca --skill orchestration --global`.
2. Observe `◇ Installation complete → ✓ Installed 1 skill`, exit 0.
3. Note the `■ Failed to install 1 → orchestration → PromptScript: PromptScript does not support global skill installation` footnote.
4. Open Orca — the skill is not installed.

## Fix direction
Generate a project-scoped install instead of `--global`, e.g. `npx skills add <repo> --skill <name> -y`, which writes `./.agents/skills/<name>/SKILL.md` where Orca discovery classifies it as a visible source. Tracked in open PR #6580 (also home-scopes so Orca sees it as a `home` source). If a global mode is ever kept, it must fail loudly (non-zero exit) when the PromptScript target is skipped.
